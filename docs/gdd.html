High Concept

A 3D roguelite where the player pilots a flying drone shepherding a forward-marching conga line of 3–5 auto-firing mechs through endless enemy waves. The drone collects pickups, triggers ultimates, and runs a 4-key WASD minigame to repair burning mechs. Each level-up offers 3 weighted boons (Hades-style) for one rolled mech. Between runs, scrap earned from waves and gold unlocks additional mech slots in the Garage.

Core Loop

Inner Loop

  1. Wave fires every 12.0 s. WaveSpawner emits 5 + (wave − 1) × 2 enemies on a 4.0 s spread, 20 u from the line.
  2. Mechs march forward at 3.0 u/s in a conga (2.5 u spacing) and auto-fire their assigned weapon.
  3. Enemies (40 HP, 4.5 u/s, 8 melee dmg) chase the nearest alive mech and attack on contact (1.0 s interval).
  4. Killed enemies drop 2 — 4 XP pickups + ~35% chance gold (1 each).
  5. Drone (14 u/s) flies the field; pickups within 5.5 u home onto it, collected at ≤ 0.9 u.
  6. XP fills the bar → time pauses → slot-machine rolls a target mech → 3 boons drawn weighted (70/25/5) → player picks one. No skip, no reroll. Resume.

Branches

  • Burn — Mech HP ≤ 45% ignites, ticks 2 dps. Drone within 5 u + Repair button → 4-key WASD minigame → success fully heals + clears burn; fail flashes red and resets sequence.
  • Death — A mech at 0 HP is lost for the rest of the run. Enemies retarget the nearest alive mech.
  • Run end — All mechs dead → DeathScreen → return to Garage with wave + floor(gold / 3) scrap. Slot 4 = 50 scrap, slot 5 = 120 scrap (must own 4 first). Save lives in user://save.json.

Hit pipeline — _apply_hit(enemy, base, pos, dir)

combo  = RunManager.combo_mult()
dmg    = base × damage_mult × combo
enemy.take_damage(dmg)

if dot_dps > 0
    enemy.apply_dot(dot_dps, 3.0)

if slow_mult < 1.0 and slow_duration > 0
    enemy.apply_slow(slow_mult, slow_duration)

if knockback_force > 0
    enemy.apply_knockback(dir.normalized() × knockback_force)

if splash_radius > 0
    for other in enemies within splash_radius:
        other.take_damage(dmg × 0.5)
        if dot_dps > 0:  other.apply_dot(dot_dps, 3.0)

Note — Garlic ult does NOT use _apply_hit; it applies damage, dot, slow, and knockback inline (always punches; stacks knockback_force on top of the 38 base).

Caps & rules

MAX_TYPES_PER_TARGET
2
MAX_STACKS_COMMON
3

Each mech (target) carries at most 2 distinct upgrade types. New types are blocked once at cap; existing stackable types remain available.

A common upgrade stacks up to 3× per slot, then drops from that target's pool.

Unique upgrades — once taken, the id is added to taken_unique_upgrades and never offered again.

Rarity weights [Common 70, Uncommon 25, Rare 5] — pool entries are weighted at draw time (Upgrades.RARITY_WEIGHTS).

Level-up flow (Hades-style)

  1. level_up fires → time pauses, UpgradePicker instances over the canvas.
  2. Slot-machine animation cycles mech portraits, lands on a random target whose pool is non-empty after available_for_target() filtering.
  3. Upgrades.pick_weighted(pool, 3) draws 3 distinct boons weighted by RARITY_WEIGHTS.
  4. Player picks one card. NO SKIP, NO REROLL.
  5. Upgrades.apply() runs the matching match upgrade.id branch (mutates weapon vars). RunManager.record_upgrade() increments stack count, emits upgrade_taken.
  6. Time resumes.

Pool filtering

  • target weapon must exist in the run
  • unique already taken → excluded
  • common at MAX_STACKS_COMMON → excluded
  • new type when at type-cap → excluded

Combo — Line-Wide Kill Chain

src/RunManager.gd · combo_mult() folded into every _apply_hit()

Constants

COMBO_MAX_STACKS
3
COMBO_DURATION
2.0 s (per-stack timer)
COMBO_PER_STACK
0.30 (+30% damage per stack)
combo_enabled
bool, default false — gates the system

Multiplier

combo_mult() = 1.0 + COMBO_PER_STACK × stacks
            →  1.0 / 1.30 / 1.60 / 1.90  (at 0/1/2/3 stacks)

Behavior

  • notify_kill() appends a stack with COMBO_DURATION; oldest is popped if size exceeds MAX_STACKS.
  • Per-stack timers tick down each frame; expired stacks are removed individually (rolling decay, not all-or-nothing).
  • Audio chime xp_collect plays per kill at pitch 1.0 + 0.18 × stack_count → audible buildup.
  • reset_run() clears stacks and disables the system.

Gap — combo_enabled is never set to true by any current upgrade in Upgrades.ALL. The system is wired but inert in-game.

Unlock rule
Only the next-up slot is purchasable. Cannot buy slot 4 without owning slot 3.

Flow

  1. Run ends → DeathScreen shows wave reached, gold collected, scrap earned.
  2. Player returns to Garage; SaveData adds the scrap delta and persists.
  3. Garage UI lists slots; the next-up slot shows its cost. Owned slots are check-marked.
  4. Buying deducts scrap and increments unlocked_slot_count. Garage emits scrap_changed and unlocks_changed.
  5. "Start Run" launches Game.tscn with the unlocked slot count, which determines line size (3 — 5 mechs).

Save format (user://save.json)

{ "scrap": int, "unlocked_slots": int }

Loaded once on autoload (SaveData), saved on each mutation. No cloud / no migration logic; a missing file resets to defaults.
